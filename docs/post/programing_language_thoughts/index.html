Programming language

Reflect

(On reading golang reflect) Reflect is not good, I feel like it tries to climb over a wall which was built by golang own. The best is not to build the wall at all.

Type safety

(On reading some c code enfocing type safety)

Type safety is good to resolve part of the programing problems (which some people think is critical to have not to introducing a subtle bugs which is difficut to debug, which I do not think), but it ignores that we have a lot more other problems which could introduce bugs, “type safe” loses great flexibility to resolve those other problems.

Type safety is not bad as long as it does not become an execuse to ignore other important issues.

Python is a good justification.
